# 2026-05-13

- 将日程应用直接数据源从 Supabase 改为项目根目录 `schedules.csv`。
- 新增 `arrows.csv` 和 Supabase `arrows` 表，用于永久保存事项之间的有方向箭头线条。
- 前端改为 React hooks 调用本地 `/api/schedules` 和 `/api/arrows`，不再直连 Supabase。
- `user_start.sh` 启动顺序改为：拉取 Supabase 到 CSV，后台启动 CSV API、Vite、每分钟同步、每小时 sidebar 生成。
- `generate_sidebar.py` 每 60 分钟按 `schedules.csv` 变化调用 Claude CLI 生成小红书风格 `sidebar.html`。
- 验证：`npm run build` 通过；`bash user_start.sh && npx playwright test schedule-list.spec.js` 通过 2 个 E2E 测试。
- 纠正 nginx 映射理解：外部 `https://hermit.dimond.top/nanjing` 已由 nginx 映射到容器 `http://localhost:18085`，因此应用内部应按根路径 `/` 服务，Vite `base` 保持 `/`，API 走 `/api/...`。
- 修复旧环境 `crypto.randomUUID is not a function`：新增 UUID fallback 工具。
- Docker 挂载核实：`18085-nanjing` 只挂载项目目录、sessions、logs、agent-config；宿主 Python/Miniconda/Homebrew 并没有挂进容器。项目 `.venv/bin/python` 是指向宿主 `/opt/homebrew/...` 的符号链接，在 Linux 容器内不可执行。
- 修复容器启动：把 npm cache 指到项目 `logs/npm-cache`，显式加入 `@rollup/rollup-linux-arm64-gnu`，避免 Vite 在 Linux ARM64 容器里缺 Rollup optional dependency。
- UI 修复：第一栏/第三栏向两侧收紧，中间栏可见箭头；箭头线可点击选中，出现小 x 后可删除，并写回 `arrows.csv`。
